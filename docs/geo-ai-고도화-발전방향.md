# GEO·AI 인용 고도화 — 사이트 발전 방향

> **결론:** GEO 관점에서 분명 좋아졌고 기본기는 꽤 갖춰졌는데, 아직 **“AI가 인용하기 최적”** 단계까지는 아니다.  
> 지금 상태는 브랜드 신뢰 정보와 FAQ는 좋아졌고, 반대로 **자사 도메인 안에서 인용할 만한 깊은 문서가 부족**하고, **크롤링 결과에 노이즈**가 보여서 손봐야 할 부분이 있다.

---

## 1. 잘 된 점

- **가치 제안이 분명함**  
  홈페이지 첫 화면에 **“보여주는 웹이 아니라, 결과를 만드는 웹”**이라는 문구가 있고, 바로 아래에 전략 기획·전환 설계·성과 측정까지 설명이 붙어 있어서 **AI가 “이 사이트가 뭘 하는 곳인지”** 파악하기 쉬워졌다.

- **신뢰 신호**  
  FAQ 섹션, 회사 주소, 대표명, 이메일, 사업자등록번호, 개인정보처리방침까지 노출되어 있어서 신뢰 신호가 괜찮다.

- **문의·상업 구조**  
  문의 페이지가 별도로 존재하고, 프로젝트 정보/예산/연락처를 받는 구조라 상업 사이트로서의 명확성이 있다.

---

## 2. 문제점 및 개선 방향

### 2.1 (최우선) 홈페이지 본문에 섞인 코드 조각

**Our Strength 구간** 크롤링 결과에 `handler.go`, `package main`, `import`, `json.NewEncoder` 같은 **Go 코드 조각**이 실제 본문 사이에 섞여 노출되고 있다.

- **영향:** GEO뿐 아니라 일반 SEO에도 악영향. AI·검색엔진이 “회사 소개 페이지인지, 코드 샘플 페이지인지” 헷갈리고, 핵심 의미가 흐려진다.
- **조치:** **우선순위 최상**으로 해당 코드 노이즈 제거.

### 2.2 페이지 제목 신호가 약함

크롤링 결과상 홈·포트폴리오·문의 페이지가 모두 상단에 **“solidweb”** 정도로만 잡혀 있어, “웹 제작 스튜디오”, “홈페이지 제작”, “Astro/Cloudflare”, “브랜딩” 같은 **주제 키워드가 제목 레벨에서 충분히 드러나지 않는다.**  
본문은 좋아져도 제목 신호가 약하면 AI 인용 후보로서 문맥 연결이 덜 선명해질 수 있다.

**권장 예시:**

- **title:** `솔리드웹 | 전환 중심 홈페이지 제작 스튜디오`
- **meta description:** `브랜딩, 홈페이지 제작, 포트폴리오, 유지보수까지. 결과를 만드는 웹을 설계하는 솔리드웹`

### 2.3 자사 도메인 내 인용 문서 깊이 부족

현재 구조는 **홈 / 포트폴리오 / 문의** 중심이고, 포트폴리오도 **개별 프로젝트에 대한 자사 도메인 내 상세 케이스 스터디가 아니라 외부 사이트로 바로 나가는 형태**다.  
즉, AI가 솔리드웹을 인용하려 할 때 **“이 회사가 어떤 방식으로 문제를 해결했고 무엇을 잘하는지”**를 solidweb.kr 내부 문서에서 충분히 읽기 어렵다.

**개선 방향:**  
서비스 상세 페이지(`/services/homepage`, `/services/landing-page`, `/services/maintenance` 등)와, **포트폴리오를 자사 도메인 내 케이스 스터디로 확장** (업종, 목표, 문제, 해결 방식, 사용 기술, 결과 등 정리).

### 2.4 FAQ 답변이 너무 일반적

FAQ 구성은 좋은데, 답변이 **“웹/앱 개발, 브랜딩, 마케팅 등”** 수준으로만 있으면 AI가 **솔리드웹만의 강점**을 뽑아 인용하기엔 정보 밀도가 낮다.

**개선 방향:**  
“소상공인/브랜드/전문가용 홈페이지”, “전환 설계”, “Cloudflare 기반 고성능 사이트”, “운영/유지보수”, “관리자 페이지 구축”처럼 **솔리드웹 특화형으로 구체적으로 재작성**.

## 3. 현재 상태 점수 (체감)

| 항목 | 평가 |
|------|------|
| 브랜드 정체성 | 좋음 |
| 신뢰 정보 | 좋음 |
| FAQ 존재 | 좋음 |
| 포트폴리오 볼륨 | 좋음 |
| 자사 도메인 내 인용 문서 깊이 | 아쉬움 |
| 크롤링 품질 | 수정 필요 |
| 메타/타이틀 선명도 | 보강 필요 |

**체감 평점:** 6.5~7/10  
방향은 맞고, **코드 노이즈 제거 + 제목/문서 구조 보강**만 하면 확 올라갈 타입.

---

## 4. 수정 우선순위 (5개)

1. **홈페이지 본문에 섞인 코드 조각 제거** — 제일 먼저.
2. **title / meta description 재작성**  
   - 예: `솔리드웹 | 전환 중심 홈페이지 제작 스튜디오`  
   - 예: `브랜딩, 홈페이지 제작, 포트폴리오, 유지보수까지. 결과를 만드는 웹을 설계하는 솔리드웹`
3. **서비스 상세 페이지 만들기**  
   - 예: `/services/homepage`, `/services/landing-page`, `/services/maintenance`
4. **포트폴리오를 자사 도메인 내 케이스 스터디로 확장**  
   - 각 프로젝트마다: 업종, 목표, 문제, 해결 방식, 사용 기술, 결과 등 상세 페이지 추가.
5. **FAQ를 솔리드웹 특화형으로 재작성**  
   - 지금보다 훨씬 구체적으로.

---

## 5. 참고 (평가 범위)

이번 확인에서는 **화면에 드러나는 크롤링 본문 기준**으로 봤고, `robots.txt`, `sitemap`, JSON-LD 구조화 데이터, `head` 메타 태그까지는 이 세션에서 확실히 검증되진 않았다.  
그래서 **“본문 기준 평가는 가능하지만, 기술 SEO/GEO 전체 합격 여부까지 단정하긴 어렵다”**고 보는 게 정확하다.
